# Changelog

## 0.14.11 — 2 July 2026

Grouped Parameter Plot zoom/interaction fixes and data-correctness hardening from the codebase review, plus a dead-code sweep.

### Grouped Parameter Plot — zoom & interaction

- **Zoom no longer resets on cosmetic changes.** Toggling markers or the legend (or changing a locked input value to compare lines) previously ran a full autoscale and threw away the user's zoom. Cosmetic toggles now apply in place, and data rebuilds preserve the viewport unless the X/Y column actually changed. ([`GroupedPlotViewModel`](DatPlotX/ViewModels/GroupedPlotViewModel.cs), [`GroupedPlotControl`](DatPlotX/Views/GroupedPlotControl.axaml.cs))
- **Ctrl+left-drag rectangle zoom** added, matching Stacked panes / the Compact surface — with a cross cursor while Ctrl is held. Plain left-drag still pans; ScottPlot's default left-pan is suppressed while Ctrl is down so it can't swallow the gesture. ([`CtrlSuppressedMouseDragPan`](DatPlotX/Views/Controls/CtrlSuppressedMouseDragPan.cs))
- **Right-drag zoom and the double-click FPS-benchmark overlay removed.** Right button is now exclusively the context menu; the menu item is renamed to **Set Scale to Default** to match Stacked mode.
- **Annotation hit-test tightened** so left-drag panning is no longer hijacked near text annotations.
- **Hover tooltips can be toggled** in Grouped mode (Tools → Show Hover Tooltips now reaches the grouped surface).

### Data correctness

- **Grouped input selection no longer blanks the plot for formatted values.** The input dropdown now carries the raw value instead of round-tripping through the (possibly lossy) formatted display string, so selecting e.g. a value shown as `0.8` still matches its rows. ([`GroupedInputParameterViewModel`](DatPlotX/ViewModels/GroupedInputParameterViewModel.cs))
- **Grouping fixed for large-magnitude columns.** The group key overflowed `long` for values above ~9.2e9 (epoch-ms timestamps, serial numbers), collapsing an entire column into one line. It now quantizes by significant digits at any magnitude. ([`GroupedDataIndexer`](DatPlotX/Services/GroupedDataIndexer.cs))
- **CSV columns no longer silently NaN out.** A column that was all-integer in the first sampled rows but fractional later was typed `int`, so later fractional values were dropped. Numeric columns now always widen to `double`. ([`CsvDataParser`](DatPlotX/Services/Parsers/CsvDataParser.cs))

### Robustness & cleanup

- **Clear Pane no longer leaves ghost event-line/annotation state.** It now removes curves only (as the confirmation promises) instead of nuking the whole plot, so global event lines and annotations survive. ([`PlotPaneViewModel`](DatPlotX/ViewModels/PlotPaneViewModel.cs))
- **Cursor allocation churn removed.** Plot surfaces now share cached `Cursor` instances instead of allocating a native cursor handle on every pointer move. ([`PlotCursors`](DatPlotX/Views/Controls/PlotCursors.cs))
- **Dead code removed** ahead of OSS release: `AutoPlotData`, `AddEventLineToAllPanes`, an unused `SynchronizeXAxes` wrapper, `FindPaneControl`, and the never-wired `GroupedAnalysisCurveSource`. MainWindow's grouped-export/crash-folder handlers are now detached symmetrically.

## 0.14.10 — 29 June 2026

Pre-OSS deep-review hardening (memory, security, config) plus the unified Statistics menu and analysis metric-picker work that landed after 0.14.9.

### Code-review fixes (memory / security / config)

- **Memory leak: plot-pane control now releases its input events.** [`PlotPaneControl`](DatPlotX/Views/PlotPaneControl.axaml.cs) subscribed eight ScottPlot pointer/keyboard events plus the context-menu `Opening` event in its constructor but only detached one of them on unload, so every pane add/remove (panes are recycled on layout changes) leaked the control through ScottPlot's event lists. Subscription is now symmetric — wired on `Loaded`, unwired on `Unloaded` (guarded against double-subscribe) — so detached panes are collectible and re-attached panes re-wire.
- **Security: stopped writing imported data to the diagnostic log path.** [`CsvDataParser`](DatPlotX/Services/Parsers/CsvDataParser.cs), [`PlotDataModel`](DatPlotX/Models/PlotDataModel.cs) and [`CurveCoordinationService`](DatPlotX/Services/CurveCoordinationService.cs) passed cell values and column names into `SafeErrorHandler.LogError`, against the security baseline ("never log row data / column names / file contents"). Harmless today (the sink is Debug-only) but a hazard the moment that handler is wired to the file logger — replaced with column/row indices and the target type only.
- **Crash safety: a fire-and-forget context-menu handler is now guarded.** `HandleAddCurvesToPane` was an `async void` invoked through an `Action` callback (not a real event handler), so an exception escaped unobserved; it now routes through `SafeInvokeAsync` like its sibling handlers.
- **Config files unified under one per-OS location.** Settings and recent-files were stored under `ApplicationData` (`%APPDATA%` / `~/.config`) instead of the app's data root — on macOS that split them from logs/crashes (`~/.config` vs `~/Library/Application Support`). Both now use a new `AppPaths.ConfigDirectory`, so settings, recent files, logs, and crash dumps live in one discoverable place. ([`AppSettingsPersistenceService`](DatPlotX/Services/AppSettingsPersistenceService.cs), [`RecentFilesService`](DatPlotX/Services/RecentFilesService.cs), [`AppPaths`](DatPlotX/Helpers/AppPaths.cs))
- **Robustness: Clear/Manage compact-curve commands can no longer go stale.** [`MainWindowViewModel`](DatPlotX/ViewModels/MainWindowViewModel.cs) now re-evaluates their `CanExecute` whenever `CompactPlot.Curves` changes, covering paths (e.g. project load) that bypass the direct mutators.
- **Security pin documented + build is warning-clean.** The `Tmds.DBus.Protocol` reference is a deliberate pin forcing a patched version over Avalonia's vulnerable transitive (advisory GHSA-xrw6-gwf8-vvr9 / NU1903) — now commented so it isn't removed as "unused." Two build warnings (CS8618, CA1861) cleared; the app project builds with 0 warnings.

### Statistics menu & analysis

- **Analysis unified into a single "Statistics" menu.** The analysis entry points are consolidated into one menu instead of being scattered. ([`MainWindow`](DatPlotX/Views/MainWindow.axaml))
- **Metric picker + slope R²/intercept.** The analysis surface gained a metric picker, and slope reporting now includes R² and the intercept.
- **User guide refreshed** for the three plot modes, the unified Statistics menu, and annotations/callouts coverage.

## 0.14.9 — 28 June 2026

Reliability/safety hardening from the pre-OSS code review — no new features.

- **Project load: GZip decompression bomb guarded (CWE-409).** [`ProjectCompressor.DecompressAsync`](DatPlotX/Services/ProjectCompressor.cs) read the decompressed `.DPX` JSON with an unbounded `ReadToEndAsync`, so a small crafted gz could expand without limit and exhaust memory. Now reads through a fixed 200 MB cap (far above any real project, which holds metadata not bulk rows) and throws `InvalidDataException` past it.
- **Memory leak: plot panes now released on dispose.** [`PlotPaneViewModel`](DatPlotX/ViewModels/PlotPaneViewModel.cs) subscribed `_curveManager.OnAxisLabelsNeedUpdate` with a lambda capturing `this` but never detached it, so disposed panes couldn't be GC'd. The delegate is now held in a field and removed in `Dispose`.
- **Crash safety: `async void` overlay refresh + fire-and-forget dialogs guarded.** [`MainWindowViewModel.RefreshInlineOverlay`](DatPlotX/ViewModels/MainWindowViewModel.cs) was `async void` called from non-event paths, so a throw after teardown crashed the app unobserved; it's now a guarded `async Task` behind a fire-and-forget wrapper with `_disposed` checks. The three fire-and-forget dialog launches (place-event-line, compact-band-limit warning, grouped-inputs prompt) route through a new `PostToUiThreadGuarded` helper that logs faults instead of letting them escape as unobserved task exceptions.
- **CSV import: cancellation plumbed through (no behavior change yet).** An optional `CancellationToken` (default-valued, all callers compile unchanged) now threads from [`DataImportService.ImportDataAsync`](DatPlotX/Services/DataImportService.cs) down through [`CsvDataParser`](DatPlotX/Services/Parsers/CsvDataParser.cs) / [`XPlaneDataParser`](DatPlotX/Services/Parsers/XPlaneDataParser.cs) to the streaming row loop, which checks for cancellation every ~16k rows. Lets a future cancel button abort a 10M-row import.

## 0.14.7 — 27 June 2026

Bug-fix release — no new features.

- **Legend: top item no longer clipped (Stacked + Grouped, ScottPlot).** The first legend entry's tallest glyphs (e.g. the d/l ascenders in "Gload") were shaved off by the legend frame. ScottPlot clips each item to `LegendRect.Contract(Padding)` while `LabelStyle.Measure` under-reserves the cap/ascender — and neither `Legend.Padding` (clip and text shift together) nor a leading newline (the `Wrapping` engine sizes every row to the tallest item, blowing out spacing) fixes it. New [`TopHeadroomLegendLayout`](DatPlotX/Helpers/TopHeadroomLegendLayout.cs) wraps the stock `Wrapping` layout and reserves a few pixels above the top item without changing per-row height; wired in [`PlotPaneFormattingService`](DatPlotX/ViewModels/PlotPane/PlotPaneFormattingService.cs) (Stacked) and [`GroupedPlotControl`](DatPlotX/Views/GroupedPlotControl.axaml.cs) (Grouped).
- **Compact: slope-line value label no longer vanishes at the band top (OxyPlot).** A full-span slope line whose endpoint lands at the top of its banded Y axis (e.g. `_dist`) had its value label drawn above the band and clipped away entirely by OxyPlot's per-band clip rect — so the line showed but its number didn't. Now detects a near-band-top endpoint and flips the label below it, keeping it inside the band. ([`CompactAnalysisOverlayHost.cs`](DatPlotX/Services/Analysis/CompactAnalysisOverlayHost.cs))
- **Analysis panel moved into the bottom pane, side-by-side with Source Data.** The Analysis Results panel now shares the collapsible bottom pane with the Source Data grid via a draggable splitter, instead of occupying separate space. Uncollapses on toggle and auto-sizes its width to content. ([`MainWindow`](DatPlotX/Views/MainWindow.axaml))
- **Grouped: empty Curves + View menus are hidden.** In Grouped mode those menus had no applicable items; they're no longer shown.
- **Internal**: Avalonia headless UI-test infrastructure added (reflection-driven XAML-load smoke test for all views); test project builds with 0 warnings (was 53).

## 0.14.5 — 19 June 2026

- **Compact arrow labels now rotate with the arrow + honor Above/Below.** Two OxyPlot-only bugs on the Compact Plot Surface (Stacked/ScottPlot was always correct):
    - **Label didn't rotate when dragging the arrowhead.** `BuildArrowLabel` gated transform-validity on `yAxis.ScreenMax.Y < yAxis.ScreenMin.Y`, but OxyPlot's `ScreenMin`/`ScreenMax` are the shared plot-area rectangle corners for *every* axis (`ScreenMax.Y > ScreenMin.Y` always), so the guard was always false and the label fell into a no-rotation, data-space-offset fallback — staying horizontal and drifting sideways as the arrow rotated. Now probes `Transform()` output for finiteness instead. ([`CompactPlotViewModel.cs`](DatPlotX/ViewModels/CompactPlotViewModel.cs))
    - **Above/Below were flipped.** "Above" added the perpendicular pixel offset (moving the label down) and "Below" subtracted it. Screen Y is downward, so Above must subtract — now matches the Stacked path.
- **Edit/Delete Annotation context-menu items now appear on Stacked + Grouped.** Right-clicking a text or arrow annotation showed the Edit/Delete items only on Compact. The press-time hit-test correctly identified the clicked annotation, but the code that makes the items visible lived solely in the `ContextMenu.Opening` handler — and Avalonia does **not** raise `Opening` when a menu is opened programmatically via `ContextMenu.Open()`, which is how Stacked and Grouped open it (ScottPlot captures the pointer and suppresses Avalonia's automatic `ContextRequested` path). Compact escaped the bug by attaching its menu to the outer control and letting the automatic event open it. Visibility is now set directly in the right-press handler before `.Open()`. ([`PlotPaneControl.axaml.cs`](DatPlotX/Views/PlotPaneControl.axaml.cs), [`GroupedPlotControl.axaml.cs`](DatPlotX/Views/GroupedPlotControl.axaml.cs))
- **Grouped: dragging an arrow end now rotates instead of translating.** `HitTestArrowAt` scored base/tip/body on one running minimum with no priority; since an endpoint also lies on the shaft, the near-zero body distance could override the endpoint hit and force a whole-arrow translate (Mode 2) instead of an end-drag (Mode 0/1 = rotate). Adopted Compact's scoring: a 14px endpoint tolerance that wins outright plus an 8px body check penalized so endpoints near the shaft still win. Stacked was already correct. ([`GroupedPlotControl.axaml.cs`](DatPlotX/Views/GroupedPlotControl.axaml.cs))
- **Slope unit fix: show Y/X rate + rate-of-climb for ftMSL / X-Plane data.** ([`SlopeMetric.cs`](DatPlotX/Services/Analysis/Metrics/SlopeMetric.cs))
- **X-Plane import: auto-detect units and strip padding underscores.** ([`XPlaneDataParser.cs`](DatPlotX/Services/Parsers/XPlaneDataParser.cs))
- **Tests**: 723 passing (added Compact arrow-label rotation + Above/Below regression coverage).
